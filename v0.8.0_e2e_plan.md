# LibreTerm v0.8.0 Plan (E2E Testing with WinAppDriver)

**Goal:** Establish a robust End-to-End (E2E) automated testing framework using Microsoft WinAppDriver to verify UI interactions and core functionality.

## 1. Test Framework Setup
*   **Tool:** WinAppDriver (Standard for Windows UI Automation).
*   **Runner:** Python with `Appium-Python-Client` and `pytest`.
*   **Target:** `build/LibreTerm.exe`.

## 2. Test Cases (MVP)
1.  **Launch:** Verify the app opens and the main window exists.
2.  **Creation:** Automate clicking "File -> New Connection", filling the form, and saving.
3.  **Verification:** Check if the new connection appears in the TreeView.
4.  **Session:** Double-click the connection (or right-click -> Connect) and verify a tab opens.
5.  **Cleanup:** Verify closing the app cleans up processes.

## 3. Implementation Steps
1.  **Environment:** Create a `tests/` directory.
2.  **Requirements:** Create `requirements.txt` (Appium, pytest).
3.  **Code:** Write `tests/test_basic.py`.
    *   Setup `webdriver.Remote`.
    *   Find elements by Name/AutomationId.
    *   Assert states.
4.  **Execution:** Create `run_tests.bat` to install deps and run pytest.

## 4. Prerequisite Check
*   Ensure WinAppDriver is running (usually port 4723).
*   Ensure Developer Mode is on (usually required for WinAppDriver).

This release focuses purely on **Quality Assurance infrastructure** rather than new features.
